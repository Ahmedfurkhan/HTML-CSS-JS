§ 01 — HTML Structure

10 Most Useful HTML Tags

These ten tags appear in nearly every page ever written. Master them and you can build almost anything — the rest is specialisation.

<div>

Division

Generic block container with zero semantic meaning. The universal grouping tag for layout, JavaScript hooks, and styling regions.

<span>

Span

Inline counterpart to <div>. Wraps a word or phrase to apply styles or attach event listeners without a line break.

<a>

Anchor

Creates hyperlinks via href. Can link to pages, sections, emails (mailto:), or files. The backbone of the web.

<p>

Paragraph

Semantic block for a run of prose text. Browsers add top and bottom margins by default, giving readable vertical rhythm.

<img>

Image

Embeds an image via src. Always include alt text for accessibility and image-search SEO. Self-closing.

<ul> / <ol>

Lists

Unordered (bullets) and ordered (numbered) lists. Items go in <li> children. Crucial for navigation menus and content lists.

<input>

Input

Creates form controls — text, password, checkbox, radio, file upload, color picker, and more. Controlled via the type attribute.

<button>

Button

A clickable element that triggers actions or form submission. Unlike <input type="button">, it can contain rich HTML content.

<section>

Section

Semantic block for a thematically grouped region. Helps screen readers and search engines understand page structure. Should contain an H2.

<meta>

Meta

Lives in <head> and describes the page to browsers, search engines, and social platforms. Invisible to users; invaluable to crawlers.

§ 02 — SEO

Key Head Tags for SEO

All eight tags below are live in the <head> of this very page. View source to inspect them. Sources: Search Engine Journal, SiteGround KB, Ahrefs (2024–2026).

<title>

Most important SEO tag. The clickable headline in Google SERPs and a primary ranking signal. Keep it under ~60 characters, include your main keyword, and match it to your page's H1. Google may rewrite it if it judges your version misleading or too short/long.

<meta name="description">

The 150–160 character blurb beneath the title in search results. Not a direct ranking factor, but a well-crafted description dramatically improves click-through rate (CTR), which indirectly signals relevance to Google. Write for humans first.

H1–H6 Heading Tags

Heading tags signal content hierarchy — not just visually but semantically. Use exactly one H1 per page (the page's topic), then H2–H6 for nested subtopics. Include relevant keywords naturally; heading content is a moderate direct ranking signal.

<meta name="robots">

Instructs web crawlers. content="index, follow" — crawl and rank this page. noindex — exclude from results (use on admin, duplicate, or staging pages). nofollow — don't pass PageRank through links on this page.

<link rel="canonical">

Solves duplicate-content problems. If the same content lives at multiple URLs (www vs non-www, with/without trailing slash, UTM parameters), canonical tells Google which is the "master" — consolidating link equity and preventing split ranking signals.

<meta name="viewport">

Not a classic ranking meta tag, but Google uses mobile-friendliness as a ranking signal. content="width=device-width, initial-scale=1.0" ensures the page renders correctly on phones — without it, Google may down-rank your site in mobile search.

Open Graph Tags (og:title, og:description, og:image)

Control how your page looks when shared on Facebook, LinkedIn, Slack, etc. A compelling OG image + headline increases social shares → drives organic traffic → earns backlinks. Indirect but meaningful SEO benefit.

<meta charset="UTF-8">

Ensures correct character encoding — preventing garbled text, especially for non-Latin scripts. Broken encoding confuses crawlers and destroys user experience, both of which hurt SEO. An indirect but foundational safeguard.

§ 05 — CSS Typography

6 Font-Size Units

Every row shows the same sentence at a different size unit — three absolute (fixed regardless of context) and three relative (scale with something else). All sized with style="" inline attributes.

  • px — Absolute · 11px Use when: you need pixel-perfect, context-independent sizing — a 1 px border, a 16 px icon, or a fixed-width UI element that must never scale.

  • pt — Absolute · 14pt Use when: targeting print stylesheets. 1 pt = 1/72 inch — meaningful on paper. On screens, prefer px or rem.
    The quick brown fox jumps over the lazy dog.
  • cm — Absolute · 0.5cm Use when: you need physical measurements for print — e.g., the fine print on a legal document must be at least 0.3 cm. Almost never on screen.
    The quick brown fox jumps over the lazy dog.
  • rem — Relative · 1.125rem Use when: sizing body text and most UI elements. 1 rem = browser root font-size (usually 16 px). Respects the user's browser font-size preference — best for accessibility.

  • em — Relative · 1.4em Use when: sizing within a self-contained component — e.g., padding: 0.5em 1em on a button so its spacing always scales with its own font-size, not the root.

  • vw — Relative · 2.4vw Use when: creating fluid hero headings that scale from phone to desktop without any media queries. clamp() is often paired with vw to set a safe min/max.

static

The factory default. The element follows the normal document flow and ignores top, left, etc. Use it when no special positioning is needed — which is most elements most of the time.

relative

Nudges an element from its own normal position without removing it from the flow (other elements still see it in its original spot). Most importantly, it creates a positioned ancestor for absolute children — a pattern used constantly in UI work.

absolute

Removed from normal flow entirely — surrounding elements ignore it. Positioned relative to the nearest ancestor that has a non-static position. Ideal for tooltips, notification badges, image overlays, and decorative shapes.

fixed

Positioned relative to the browser viewport — stays perfectly still as the user scrolls. Used for persistent UI: cookie banners, floating action buttons, live chat widgets, and — as demonstrated here — the site footer and (as sticky) the navbar.

Bonus — sticky: The <nav> at the top of this page uses position: sticky; top: 0 — a fifth mode. It behaves like relative until it would scroll off-screen, then locks in place like fixed. Perfect for navigation bars; try scrolling to watch it stick.
